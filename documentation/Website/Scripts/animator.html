getJSON(String path, Object json)

Takes the path of a JSON file and parses it, creating an object of the json text.
The JSON object is then read to create HTML elements out of the objects needed to output the Sequence Diagram.

In the context of the DIT029 Format, we can take the example of the Processes (Objects in the SSD).
The function iterates through the array and creates a "div" element and appends it as a child of the Animation Panel in animationPage.html.

Furthermore, it creates another "div" and appends it to the previous "div" in order to later style it into a UML lifeline.
Finally, after very iteration, it will check whether or not the element being created is the last one or not.
If not, the function will create another "div" and append it as a child of the main panel. This is done to simplify the CSS layout.
N.B: The three constant variables (see below) are used to change the name of the classes of the elements in HTML. This has consequences at runtime and within the CSS implementation, do not change this unless you know what you are doing.

const processDivClassName = "processes";
const betweenDivClassname = "betweenProcesses";
const lifelines = "lifeLine";

createNodes(object, frameToAppend)

This function is responsible for parsing the JSON File nodes in its entirety. It takes two parameters, a JSON Object (this could also be an array) as well as the frame to append the nodes to.
It is called recursively after each node creation and stops once the parsing is done. The initial JSON must have at least one "send" node otherwise the function will hang
